# Анализ и прогнозирование ценообразования на образовательные услуги

**Автор:** Горячевская Екатерина Николевна<br>
**Тема ВКР:** «Анализ и прогнозирование ценообразования на образовательные услуги»

[![Открыть в Google Colab](https://colab.research.google.com/assets/colab-badge.svg)](https://colab.research.google.com/github/Kathrin-23/muiv_costs/blob/main/ml/train_model.ipynb)

Веб-приложение на Flask загружает и анализирует сведения о стоимости
образовательных услуг, сравнивает модели машинного обучения и сохраняет прогнозы
с указанием алгоритма, который их сформировал.

## Данные и воспроизводимость

Синтетический набор заменен воспроизводимой сборкой из проверяемых открытых
источников:

- официальные открытые данные Роспатента о платных образовательных услугах РГАИС;
- открытые данные РУДН о численности студентов по направлениям подготовки;
- открытый API Всемирного банка с индексом потребительских цен РФ.

Подробное происхождение и правила расчета производных признаков описаны в
[`data/SOURCES.md`](data/SOURCES.md). Каждая строка исходного файла содержит URL
источника и номер исходной записи.

Повторная сборка:

```bash
python data/build_dataset.py
```

Команда формирует:

- `data/raw/educational_services_prices.csv` — исходный набор с полями
  происхождения;
- `data/processed/prepared_dataset.csv` — очищенная таблица для ML.

В текущей версии используется 691 наблюдение из пяти публикаций Роспатента за
2017–2024 годы. Случайная генерация значений в скрипте отсутствует.

## Сравниваемые модели

Одна команда последовательно настраивает и обучает:

- Random Forest (`RandomForestRegressor`);
- Gradient Boosting (`GradientBoostingRegressor`);
- гребневую регрессию (`Ridge`).

```bash
python ml/train_model.py
```

Для каждой модели рассчитываются MAE, RMSE, MAPE и R² на обучающей,
валидационной и тестовой выборках. Параметры подбираются через GridSearchCV,
а стабильность оценивается пятикратной кросс-валидацией. Итоговая модель
выбирается по минимальному RMSE на валидационной выборке, без использования
тестовой выборки при выборе.

Текущий эксперимент выбрал Ridge:

| Модель | Validation RMSE | Test RMSE | Test R² |
|---|---:|---:|---:|
| Random Forest | 43 255,33 | 42 768,16 | 0,7741 |
| Gradient Boosting | 36 354,00 | 35 060,06 | 0,8482 |
| Ridge | 25 561,46 | 22 196,23 | 0,9391 |

## Артефакты обучения

Модели сохраняются независимо:

- `ml/model_random_forest.pkl`;
- `ml/model_gradient_boosting.pkl`;
- `ml/model_ridge.pkl`.

Результаты эксперимента:

- `ml/metrics_random_forest.json`;
- `ml/metrics_gradient_boosting.json`;
- `ml/metrics_ridge.json`;
- `ml/model_registry.json` — единый реестр и выбранная модель;
- `ml/model_comparison.csv` — сравнительная таблица;
- `ml/diagnostics/` — графики и JSON важности признаков.

## Загрузка набора данных

На странице «Датасеты» доступны оба способа:

- локальный CSV/XLSX;
- прямая HTTP(S)-ссылка на CSV/XLSX.

Скачивание по URL выполняется потоково с ограничением 16 МБ. После загрузки
используется общая проверка обязательных полей и очистка данных. Локальная
загрузка файла сохранена без изменений.

## Прогнозирование

На странице прогноза пользователь выбирает одну из сохраненных моделей. Название
алгоритма записывается в `ForecastResult.model_name` вместе с ценой, входными
параметрами и рекомендацией. Модель отображается в личном кабинете, отчетах и
экспортах. Для прогноза из карточки услуги автоматически используется модель,
выбранная сравнительным экспериментом.

## Быстрый запуск

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python main.py
```

В Windows:

```bat
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python main.py
```

Приложение откроется по адресу <http://127.0.0.1:5000>.

## Тесты

```bash
python -m pytest -q
```

Тесты проверяют локальную и URL-загрузку, сравнительное обучение и сохранение
трех моделей, метрики трех выборок, кросс-валидацию, выбор модели при прогнозе,
запись `model_name`, отображение сравнения и структуру выполненного Colab.

## Основные модули

- `app/ml_service.py` — подготовка данных, обучение, диагностика и прогноз;
- `app/file_service.py` — локальная и URL-загрузка CSV/XLSX;
- `app/routes.py` — маршруты и взаимодействие веб-интерфейса с сервисами;
- `app/models.py` — модели SQLAlchemy;
- `ml/train_model.py` — консольный запуск сравнительного эксперимента;
- `ml/predict.py` — пример прогноза автоматически выбранной моделью;
- `ml/train_model.ipynb` — полноценный Colab-эксперимент.

Технологии: Python, Flask, Flask-SQLAlchemy, pandas, scikit-learn, joblib,
Matplotlib, SQLite, openpyxl, python-docx и pytest.
